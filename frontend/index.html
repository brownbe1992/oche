App configuration and integrations. Admin login required.

Logged in as
Admin accounts

Admin accounts can access this Settings page and manage player PINs. There must always be at least one.

Admin login backoff: a wrong password never permanently locks this account out — each consecutive failure past the grace attempts below schedules a longer wait (doubling each time, up to the cap) before the next attempt is allowed. The correct password always works immediately once that wait has passed.

Player PINs

A PIN-protected player must have their PIN entered before they can be added to a game — useful for stopping others from playing as you. Players without a PIN can be picked by anyone.

Scoring

Which dart entry method a new game opens with. Either can still be switched per-session from the scoring screen itself. Pad is the app's accessible input path — ordinary focusable buttons, no dartboard shape or precise tap-target aiming required — worth setting as the default for a low-vision or motor-impaired player.

Accessibility

Switches the app's red/green double and treble colors (dartboard rings, pad buttons, win/bust status, live scoreboard) to a colorblind-friendly blue/orange palette. Applies to this device and the live scoreboard.

For a low-vision or motor-impaired player, see Scoring → Default input above — Pad mode is the app's accessible input path (plain focusable buttons, no dartboard shape or precise tapping required).

Voice Announcements

Spoken call-outs on the live scoreboard (`/display`) using the browser's built-in speech synthesis — no server involvement, no external service. Off by default. Each call-out can be turned on or off individually once voice is enabled.

Shareable Moments

A short line printed on every shareable card (180s, Big Fish, nine-darters, match wins, personal bests). Update this once you have a website or social handle to point people at.

Data Collection

Records the exact moment each dart is tapped, in addition to existing per-visit data — useful for analyzing throwing pace, e.g. correlating with scoring or dart weight. Off by default since most setups won't need it.

Live Scoreboard

Controls what the read-only scoreboard (the "Scoreboard ↗" link, meant for a second screen or TV by the board) shows. Pick the density that fits the screen it'll run on — each layout still scales to fill the screen.

Checkout suggestions, achievement flashes, and the H2H score panel always show, regardless of layout.

Daily Challenge

Each player gets one Daily Challenge attempt per calendar day. Resetting an attempt deletes it and every stat recorded during it (the game, turns, and darts), so the player can retake that day's challenge with a clean slate. Badges earned during the attempt are kept.

Server Errors

The most recent server-side failures (up to 500), newest first — the same record kept for a self-hosted setup without shell/docker logs access. Ordinary player mistakes (a bad login, an invalid PIN) don't appear here; only genuine server errors do.

Backups

A backup is a consistent, point-in-time snapshot of the whole database — every player, game, and stat. Take one on demand below, or schedule backend/backup.js via host cron (see README's "Backups" section). Restoring replaces the entire live database and requires your admin password again, even though you're already logged in — it stages the file and then asks you to restart the app; it doesn't restart itself.

The file is checked (header + integrity check) before anything is replaced — an unrelated or corrupt file is rejected with no changes made.

Data Export

Download a complete JSON export of every player, game, and stat in the database — it's your data, and you can always take it with you. Admin credentials and session data are never included.

Merge Players

Combine two player records that are really the same person — a typo'd second account, or someone added twice under different names. One player keeps their name and settings and absorbs the other's entire history (games, stats, badges, equipment); the duplicate is then deleted. You'll see a full preview of exactly what will move before anything happens, and the merge refuses to run if the two records conflict (e.g. they've played each other). This can't be undone.

Danger Zone

Permanently deletes every player, game, and stat in the database. Admin accounts and these settings are kept. Meant for clearing out test/dev data, not everyday use.

Export Player

Download one player's full history as JSON — every game they've played, including opponents' turn data from those same games (so results like "Ben beat Alaina" stay intact when moved to another server), plus a minimal identity record for each opponent. Or grab a simpler CSV of just their own stats for a spreadsheet. Admin only.

Import a player

Import a player export downloaded from this or another Oche server. A player is matched to an existing one by its export identity, not just by name — a coincidental same-name player already on this server is never merged with it. Importing the same file twice is safe: already-present games are skipped, not duplicated.
